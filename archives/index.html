    Hack to the Basics (HttB)

    So what is this all about?

    Hack to the Basics (HttB) is a Github hosted blog created with Atom, Git, NodeJS, Hexo and 3-Hexo. It has been built and sculpted with the primary intention of being used as a writing outlet for its author, K. Bradley. - One that will not only allow him to track the broad range of self-taught skills he is dedicating time to - but also as a way to give back to others who may find the topics of interest.

    The blog title Hack to the Basics, which is a run on the phrase ‘Back to the Basics’, was chosen for two reasons:

    • A lot of the content found within will focus on the computer security / ‘hacking’ field
    • The content is written with the intent that it will be fairly easy to ‘digest’

    Given the primary intention is to promote self-taught skills in the form of genuine content, this blogs secondary intention is to serve as an online portfolio.

    Navigating

    The blog is split into three main sections when visiting from a computer, and two main sections if visiting from mobile. A simple explanation on navigating these sections is found below.

    1. Left Navigation Panel / Top Panel


    The dark blue panel located on the left side of the screen for those visiting from a computer could be considered the main navigation panel. By clicking on my picture, you will be directed to a page about myself. Underneath my name are icons which will link you to various social media accounts I own.

    Beneath the icons is the main menu & sub menus. Please note that the main menu & sub menu does not exist when visiting from a mobile device.

    Clicking on a menu category will open a list of more specific sub categories. After clicking on one of these sub categories, a list of articles fitting this category will load in the Article Panel.

    2. Article Panel


    Directly to the right side of the Navigation Panel is the Article Panel. On mobile, this is shown as the lower 70% of the page. This is the panel articles will populate in depending on:

    • The menu / sub menu category chosen
    • Searched tags

    So what are tags? Each article I create is categorized and sub categorized. Tags are used for two reasons:

    • To add a method to search articles on mobile
    • To add a more specific method to searching

    All articles will be given tags with the same category names, but built upon even farther. If I write an article on programming in python, it would make sense you could find this in the Computers -> Programming sub category. As such, it will also be tagged as Computers and Programming. Additionally, it will be tagged as Python.

    The rationale, as noted above, is as articles become more abundant, it will make navigating them easier. Within the article panel, you will notice at the top right corner there is a tags button. Clicking that on computer or mobile will open a Search box. You can then search through the current category articles with tags of your choice.

    A small arrow will show midway down on the right side of the Article Panel when hovering over it with a mouse. Clicking this arrow will minimize both the Navigation Panel and Article Panel, dedicating the entire screen to the Content Window.

    Note that because mobile does not have a menu, the blog defaults the category chosen to all. So searching for articles on mobile via the tag search will always search all posted articles. On a computer, it will only search the currently selected category or sub category.

    3. Content Window


    The content window will display any content you have chosen by means explained above. Notice the two icons on the top-right corner. The top icon will expand the content window.

    The second icon will generate a Table of Contents for the currently loaded article. You can use this to skip around each article more efficiently.
